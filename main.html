A Serialization mechanism of structured data ### Why? - Language-neutral - Platform-neutral - Less ambiguous with explicit data types - Fast serialization - Lightweight --- class: middle, center # Protobuf vs. JSON --- ### Test Data: Array of "tickers" ### Protobuf Definition: ```protobuf syntax = "proto3"; package sample; message Test { string query = 1; int32 page_number = 2; int32 result_per_page = 3; repeated Ticker tickers = 4; } message Ticker { string name = 1; float value = 2; } ``` --- ### JSON Example: ```json { "query": "myQuery", "page_number": 42, "result_per_page": 100, "tickers": [ { "name": "rPs", "value": 9.768923 }, { "name": "WEo", "value": 6.067048 } ] } ``` --- class: table # Results: ### Raw JSON vs. Protobuf

--- .pull-left[ # gRPC ### What? A RPC (**R**emote **P**rocedure **C**all) framework ### Why? - HTTP/2 Transport - Simple service definition - Works across languages and platforms - Bi-directional streaming and integrated auth ] ![](https://grpc.io/img/landing-2.svg) --- # How Does It Compare to **REST**   | | gRPC | REST | |--- |:---: |:---: | | Transport | HTTP/2 | HTTP 1.1 | | Transfer | Binary | Text | | Serialization | Protobuf | JSON | | Streaming | Yes | No | | Error Handling | Descriptive exception in client call | HTTP Status code | | Schema validation | Automatically according to proto file | Manually by server controller | | Decoupling | Business code is always separated | Can be separated | | Debugging | Specific client or through a gateway | Easily via curl or postman | --- class: middle, center # Steps to create a service --- ### **Step 1**: Create a definition of your service using the **proto** language: ```protobuf syntax = "proto3"; package riskified; message GreetRequest { string name = 1; } message HelloReply { string message = 1; } service Greeter { rpc Greet(GreetRequest) returns (HelloReply) {} } ``` --- ### **Step 2**: Run the `protoc` tool to generate the service stub from the **ptoto** definition you've just wrote in the language you need (java, ruby, etc.) ```bash protoc \ --ruby_out=$RUBY_DIR \ --java_out=$SCALA_DIR \ $SRC_DIR/sample_service.proto ``` --- ### **Step 3**: Create the service implementation (the actual logic): ```ruby class GreeterServer < Riskified::Greeter::Service def say_hello(hello_req, _unused_call) Helloworld::HelloReply.new(message: "Hello #{hello_req.name}") end end ``` --- ### **Step 4**: Create a command to run the server: ```ruby def main s = GRPC::RpcServer.new s.add_http2_port('0.0.0.0:50051', :this_port_is_insecure) s.handle(GreeterServer) s.run_till_terminated enda ``` --- class: middle, center # Who's Using ._3col[![Left-aligned image](https://grpc.io/img/square-icon.png)] ._3col[![Left-aligned image](https://grpc.io/img/netflix-logo.png)] ._3col[![Left-aligned image](https://grpc.io/img/cisco.svg)] ._3col[![Left-aligned image](https://grpc.io/img/coreos-1.png)] ._3col[![Left-aligned image](https://www.google.co.il/images/branding/googlelogo/2x/googlelogo_color_272x92dp.png)] ._3col[![Left-aligned image](https://3on3f11f1a042q4pv95221y1-wpengine.netdna-ssl.com/wp-content/uploads/2017/06/namely-logo-min.png)] --- class: middle, center # **Q** & **A** --- class: middle, center # END
